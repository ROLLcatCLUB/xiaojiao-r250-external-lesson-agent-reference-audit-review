# Lesson Bundle Output Chain Reference

## Reference

Claw-ED is the strongest reference for future output package shape. The useful idea is not "generate everything now"; it is that a teacher-prep product eventually needs a coherent bundle of teacher-facing, student-facing, display-facing, assessment, and differentiation artifacts.

## Future Xiaobei Bundle Shape

Future R95/R230/R240 planning can use this output chain:

| Artifact | Purpose | Current/future source |
| --- | --- | --- |
| `lesson_design` | Main teaching design, still grounded in `single_lesson_template` | R201/R220 now |
| `teacher_execution_guide` | Teacher-facing action script and decision points | Future R210/R230 |
| `student_worksheet` | Student handout, task sheet, or learning sheet | Future R95 |
| `slide_storyboard` | Big-screen/PPT page plan, not necessarily PPTX first | Future R230 |
| `assessment_rubric` | Exit ticket, rubric, formative checks | Future quality sentinel/R95 |
| `differentiation_support` | IEP/ELL/不同层次支架 and alternatives | Later, after source policy |
| `classroom_timeline` | Time-boxed classroom flow | Future R230/R250 |
| `teacher_review_notes` | What changed, what needs confirmation, why | R201M-style teacher gate |

## Must Differ From Claw-ED

Our chain should add safeguards before any bundle export:

- Every generated artifact keeps a source ledger or source confidence marker.
- Model output stays candidate until teacher review accepts it.
- Preview happens before formal apply.
- Export package is downstream of readonly route confirmation, not a substitute for it.
- Right rail, bottom Xiaojiao, and big screen remain separate surfaces until explicitly bound.

## Practical Rule

R95 should not be reopened because Claw-ED has a rich bundle. R95 becomes appropriate only after:

1. `single_lesson_template` is stable enough for structured artifact projection.
2. Center-body readonly route has canary evidence.
3. Teacher review gate can distinguish baseline, model candidate, accepted preview, rejected, and deferred.
4. Big-screen/slide storyboard has a route surface contract.
